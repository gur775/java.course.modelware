# Практика #2

## Что означает использование final при в данном объявлении?
```java
try {
    ...
} catch (final Exception e) {
    ...
}
```
Значение ```final``` в данном объявлении запрещает изменение ссылки ```e``` внутри блока ```catch```. Нельзя присвоить параметру исключения новое значение.

Без final (можно, но не рекомендуется):

```java
catch (Exception e) {
    e = new Exception("Новое исключение"); // Так можно без final
}
```
С final:

```java
catch (final Exception e) {
    // e = new Exception(); // Ошибка компиляции
    e.printStackTrace();    // Можно только читать
}
```
Это нужно для безопасности в многопоточной среде.
Может использоваться в лямбда-выражениях внутри catch-блока.
