# Практика #1

### Объясните для каких целей используются основные классы исключений.
```
java.lang.Object
    └── java.lang.Throwable
        ├── java.lang.Error
        └── java.lang.Exception
             └── java.lang.RuntimeException
```
### Throwable
Корневой класс всех исключений и ошибок. Содержит стек-трейс и сообщение об ошибке.

### Error
Серьёзные проблемы, которые приложение не должно пытаться обрабатывать. Возникают на уровне JVM.

Примеры:

OutOfMemoryError — не хватает памяти

StackOverflowError — переполнение стека вызовов

VirtualMachineError — ошибка виртуальной машины

### Exception (проверяемые исключения - checked)
Исключения, которые компилятор заставляет обрабатывать (try-catch или throws). Проблемы, которые могут возникнуть при корректной работе программы.

Примеры:

IOException — ошибка ввода-вывода

SQLException — ошибка БД

ClassNotFoundException — класс не найден

InterruptedException — прерывание потока

### RuntimeException (непроверяемые исключения - unchecked)
Исключения, возникающие из-за ошибок программиста. Компилятор не требует их обработки.

Примеры:

ArithmeticException — деление на ноль

NullPointerException — вызов метода у null

IllegalArgumentException — неверный аргумент

ClassCastException — неверное приведение типов
