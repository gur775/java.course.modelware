# Практика #2

## Что случится при вызове метода method() объекта класса C, если переменная a и метод method() не будут определены в классе B, а будут определены только в классе A?
```java
class A {
    int a;
    void method() {
        ...
    }

}

class B extends A {
    ...
}

class C extends B {
    ...
    void method() {
        ...
        int a = super.a;
        super.method();
    }
    ...
}
```
Всё будет работать нормально.


```super.a``` обратится к полю ```a``` из класса ```A```, так как в классе ```B``` его нет

```super.method()``` вызовет метод ```method()``` из класса A, так как в классе B его нет

Вывод программы будет:

```text
Метод из C
Метод из A
```
Слово ```super``` ищет поля/методы вверх по иерархии наследования, пока не найдёт. Оно не обязано обращаться именно к непосредственному родителю — можно "пропустить" класс B к классу A.





# Практика #3

## Перепешите следующий код с использованием конструкции this(). Помните, что в каждом конструкторе вызов this() должен быть единственным и первым среди всех операций.
```java
class A {
    int a;
    int b;
    int c;
    int z;

    public A() {
        z = 1;
    }

    public A(int a) {
        this.a = a;
        z = 1;
    }

    public A(int a, int b) {
        this.a = a;
        this.b = b;
        z = 1;
    }

    public A(int a, int b, int c) {
        this.a = a;
        this.b = b;
        this.c = c;
        z = 1;
    }
    ...
}
```
### Переделанный код
```java
class A {
    int a;
    int b;
    int c;
    int z;

    // Базовый конструктор с минимальным количеством параметров
    public A() {
        this(0); // вызов конструктора A(int)
    }
    
    public A(int a) {
        this(a, 0); // вызов конструктора A(int, int)
    }
    
    public A(int a, int b) {
        this(a, b, 0); // вызов конструктора A(int, int, int)
    }
    
    public A(int a, int b, int c) {
        this.a = a;
        this.b = b;
        this.c = c;
        this.z = 1; // только здесь инициализируем z
    }
}
```
