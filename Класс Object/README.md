# Практика #1
## Самостоятельно придумайте класс и переопределите для него метод equals(). Каким соглашениям должна следовать реализация этого метода?
### Соглашения метода equals()
Реализация метода equals() должна следовать правилам, описанным в java.lang.Object:

#### Рефлексивность
Для любого ненулевого значения x, вызов ```x.equals(x)``` должен возвращать ```true```.

#### Симметричность
Для любых значений x и y, если ```x.equals(y)``` возвращает true, то и ```y.equals(x)``` должен возвращать ```true```.

#### Транзитивность
Если ```x.equals(y)``` возвращает ```true```, а ```y.equals(z)``` возвращает ```true```, то ```x.equals(z)``` также должен возвращать ```true```.

#### Согласованность
Повторные вызовы ```x.equals(y)``` должны возвращать одно и то же значение (если информациядля сравнения не изменилась).

Для любого ненулевого значения x, вызов ```x.equals(null)``` должен возвращать ```false```.

#### Зачем переопределять hashCode вместе с equals?
Если этого не сделаеть, класс будет неправильно работать в коллекциях на основе хеширования, таких как HashMap или HashSet. Если два объекта равны согласно ```equals()```, то их ```hashCode()``` должен быть одинаковым. Иначе коллекция не сможет правильно найти объект, который в ней уже находится.
