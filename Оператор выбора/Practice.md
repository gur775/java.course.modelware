# Практика #2
## Что произойдет, если в некоторых частях case будет отсутствовать break, как в следующем примере?
```java
case 1:
    i = 1;
    break;
case 2:
    i = 2;
case 3:
    i = 3;
    break;
```
Программа выдаст результат: ```i = 3```

При ```value = 2```:

Программа заходит в ```case 2```

Выполняет ```i = 2```

Нет ```break``` → происходит проваливание (fall-through)

Выполнение продолжается в ```case 3``` (условие не проверяется)

Выполняет ```i = 3```, значение перезаписывается

Встречает ```break``` → выход из ```switch```
