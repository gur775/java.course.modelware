# Практика #3

### Приведите пример и объясните, когда использование аннотации @Override помогает обнаружить ошибку.
@Override помогает:

Компилятор проверяет, что метод действительно существует в суперклассе.

При рефакторинге (переименовании методов) сразу видно ошибку.

Предотвращает случайную перегрузку вместо переопределения.

Пример: метод родителя изменили (переименовали, изменили сигнатуру), а в подклассе забыли обновить.

```java
class ParentV1 {
    public void saveData() {
        System.out.println("Сохранение данных");
    }
}

class ChildV1 extends ParentV1 {
    @Override
    public void saveData() {  // работает
        System.out.println("Сохранение с проверкой");
    }
}

// метод переименовали
class ParentV2 {
        public void save() {  // метод переименован
            System.out.println("Сохранение данных");
    }
}

class ChildV2 extends ParentV2 {
    @Override
    public void saveData() {  // ошибка saveData() не существует в ParentV2
        System.out.println("Сохранение с проверкой");
    }
    // Компилятор выдаст: method does not override or implement a method from supertype
}
```
Другая ошибка: опечатка в сигнатуре

```java
class Parent {
    public void process(int value) { }
}

class Child extends Parent {
    @Override
    public void process(Integer value) { }  // ОШИБКА! int != Integer

    // @Override
    public void process(int value, int flag) { } // без @Override - просто перегрузка
}
```
